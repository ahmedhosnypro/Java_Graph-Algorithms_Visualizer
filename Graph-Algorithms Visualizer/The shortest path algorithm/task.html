Description

In this stage, we will implement the shortest path functionality. It is a path between vertices in a graph that produces the least possible sum of the edge weights. We will work on one of the most popular pathfinding algorithms â€” Dijkstra's single-source shortest path algorithm.

Objectives

In this stage, you need to implement the following:

  1. Add the Dijkstra's Algorithm JMenuItem in the Algorithms menu and name it Dijkstra's Algorithm;
  2. Keep the algorithms from the previous stage;
  3. The algorithm should produce a list of <Vertex>=<Cost> pairs separated by a comma in the display JLabel . For example: A=1, B=2, C=3, D=4. This means that the shortest path of A, B, C, D from the selected source vertex is 1, 2, 3, 4 respectively.

At this point, your application code architecture should be flexible enough so that you could easily extend its functionality by adding a new algorithm. The classes of your application should follow the Open-close principle in Object-Oriented design. We advise you to devise the code architecture in a way that allows you to add only one class for a new algorithm.
